Mailing Lists

There are six GTK+ mailing lists.

gtk-list@gnome.org

    This is the main mailing list, and probably the one you should ask questions about GTK+. The topic of this list is general topics related to GTK+, including everything from discussion of proposed new features to questions about compiling GTK+.

    To subscribe to this list, send "subscribe" in the subject to gtk-list-request@gnome.org or visit the list information page.

    This list is archived.

gtk-app-devel-list@gnome.org

    Discussion on this list is oriented toward developing applications with GTK+. This has been a lower traffic list which is good for asking simple GTK+ questions.

    To subscribe to this list, send "subscribe" in the subject to gtk-app-devel-list-request@gnome.org or visit the list information page.

    This list is archived.

gtk-devel-list@gnome.org

    This list is for developers of GTK+ to discuss code. General GTK+ questions should not be asked on this list, as that is more appropriate for gtk-list. Requests for new features should not be sent to this list, as that is also more appropriate for gtk-list.

    To subscribe to this list, send "subscribe" in the subject to gtk-devel-list-request@gnome.org or visit the list information page.

    This list is archived.

gtk-doc-list@gnome.org

    This list is for the coordination of GTK+ documentation efforts.

    To subscribe to this list, send "subscribe" in the subject to gtk-doc-list-request@gnome.org or visit the list information page.

    This list is archived.

gtk-i18n-list@gnome.org

    This list is intended for the discussion of the internationalization and localization of GTK+ itself and of GTK+ applications or visit the list information page.

    To subscribe to this list, send "subscribe" in the subject to gtk-i18n-list-request@gnome.org.

    This list is archived.

gtk-perl-list@gnome.org

    This list is intended for the discussing the usage of GTK+ with Perl.

    To subscribe to this list, send "subscribe" in the subject to gtk-perl-list-request@gnome.org or visit the list information page.

    This list is archived.
